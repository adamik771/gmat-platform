# Module 5: Data Insights

## Learning Objectives
- Understand all five DI question types and the specific approach for each
- Master the Data Sufficiency elimination framework
- Build efficient strategies for table and graph questions
- Learn to manage information across multiple tabs in MSR questions

---

## The DI Section Overview

20 questions in 45 minutes (~2:15 per question). Five question types:
- **Data Sufficiency (DS):** ~5-6 questions
- **Multi-Source Reasoning (MSR):** ~3-4 questions
- **Table Analysis:** ~3-4 questions
- **Graphics Interpretation:** ~3-4 questions
- **Two-Part Analysis:** ~3-4 questions

**Important:** You have an on-screen calculator for DI. Use it for complex arithmetic but don't become dependent on it — estimation is still faster for many questions.

## Data Sufficiency

DS is unique. You're not solving for an answer — you're determining whether you CAN solve with the given information.

### The Five Answer Choices (Always the Same)
- **(A)** Statement (1) ALONE is sufficient, but statement (2) alone is not sufficient.
- **(B)** Statement (2) ALONE is sufficient, but statement (1) alone is not sufficient.
- **(C)** BOTH statements TOGETHER are sufficient, but NEITHER statement ALONE is sufficient.
- **(D)** EACH statement ALONE is sufficient.
- **(E)** Statements (1) and (2) TOGETHER are NOT sufficient.

### The Elimination Framework

**Step 1:** Evaluate Statement (1) alone.
- If sufficient → eliminate B, C, E → answer is A or D
- If NOT sufficient → eliminate A, D → answer is B, C, or E

**Step 2:** Evaluate Statement (2) alone.
- If Statement (1) was sufficient and Statement (2) is sufficient → **D**
- If Statement (1) was sufficient and Statement (2) is NOT sufficient → **A**
- If Statement (1) was NOT sufficient and Statement (2) is sufficient → **B**
- If Statement (1) was NOT sufficient and Statement (2) is NOT sufficient → evaluate together → **C** or **E**

### DS Traps to Watch

**Trap 1: "Solving" when you shouldn't.** You don't need to find the actual value. You just need to know IF you could find it. If a statement gives you a unique value, it's sufficient — even if you don't compute the value.

**Trap 2: Forgetting constraints.** The question stem often has constraints (e.g., "x is a positive integer"). These apply to both statements and can make a statement sufficient when it otherwise wouldn't be.

**Trap 3: The "obviously C" trap.** If both statements individually seem insufficient, many people jump to C. But sometimes they're also insufficient together — leading to E. Always check.

## Multi-Source Reasoning

MSR questions present information across 2-3 tabs and ask you to synthesize.

### The MSR Approach

**Step 1: Skim all tabs first (60-90 seconds).** Understand what each tab contains and how they relate. Don't read every word — just know what's where.

**Step 2: Read the question.** Identify which tabs contain relevant information.

**Step 3: Cross-reference.** Most MSR answers require combining data from multiple tabs. One tab might give you a number, another gives context for what that number means.

### MSR Tips
- Tab order matters — later tabs often reference or modify information from earlier tabs
- Pay attention to dates, conditions, and qualifiers in each tab
- If a question asks "Can you determine X?", check if all necessary data points are available across all tabs
- Don't assume information that isn't stated — stick to what's in the tabs

## Table Analysis

You'll see a data table (sortable by column) and 2-3 Yes/No statements.

### The Table Approach

**Step 1: Scan the table structure.** Note column headers, units, and the number of rows.

**Step 2: For each statement,** determine which column(s) are relevant.

**Step 3: Sort mentally or scan.** For statements about "the highest," "the lowest," or rankings, imagine sorting by the relevant column.

### Table Analysis Tips
- Read column headers carefully — "Revenue ($M)" is different from "Revenue Growth (%)"
- Watch for statements that require comparing two columns (e.g., "More X than Y" requires looking at both columns)
- "At least" means ≥, "more than" means >. This distinction can change the answer.
- When in doubt, check each row against the statement systematically

## Graphics Interpretation

You'll see a description of a graph or chart and answer fill-in-the-blank questions.

### Graph Reading Framework

**Step 1: Read the title and axis labels.** Understand what's being measured and the units.

**Step 2: Note the scale.** Is the y-axis starting at 0 or some other number? Non-zero baselines can make differences look larger than they are.

**Step 3: Answer the question.** Use the graph to find specific values, trends, or relationships.

### By Graph Type

**Line Graph:** Focus on trends (increasing, decreasing, stable), peaks, and valleys. The slope tells you the rate of change.

**Bar Chart:** Compare heights for individual values. For stacked bars, look at both individual segments and totals.

**Scatter Plot:** Look for correlation direction (positive, negative, none), outliers, and clustering patterns.

**Pie Chart:** Focus on relative proportions. Each slice represents a percentage of the whole.

### GI Tips
- "Closest to" questions allow estimation — don't spend time on exact calculations
- Pay attention to whether the question asks about a value, a change, or a percentage
- When interpolating between data points, use the halfway point as a reference

## Two-Part Analysis

You'll see a scenario with a two-column answer grid. You need to select one answer in each column, and both must be correct.

### Two-Part Analysis Types

**Quantitative:** Two related math problems. Often, solving one part gives you information for the other.
**Example:** "Find the speed of Train A and Train B if they meet after 3 hours traveling toward each other from 480 km apart."

**Logic-based:** Two statements that must be logically consistent.
**Example:** "Identify the assumption and the conclusion from a list of statements."

### TPA Approach
1. Read the scenario carefully
2. Understand how the two parts relate to each other
3. Try to solve one part first — it often constrains the options for the other
4. Verify that both selections are internally consistent

## DI Timing Strategy

Budget your time across question types:
- **DS:** 2:00 per question — these should be fastest since you're evaluating sufficiency, not calculating
- **MSR:** 2:30 per question — the tab-reading takes time on first exposure, then questions are faster
- **Table:** 2:00 per question — straightforward if you read the table systematically
- **GI:** 2:00 per question — estimation skills save time
- **TPA:** 2:30 per question — these tend to be the most time-consuming

If any question takes more than 3 minutes, flag it, make your best guess, and move on.

---

## Common DI Mistakes

1. **Spending too long on DS.** If you can't determine sufficiency within 2 minutes, you're probably overcomplicating it. Go with your instinct.
2. **Not reading all MSR tabs.** Some students read only the first tab and try to answer. The answer almost always requires cross-referencing.
3. **Misreading table data.** Confusing rows, reading the wrong column, or missing units.
4. **Ignoring graph scale.** A bar that looks twice as tall might only be 10% higher if the y-axis starts at 90.
5. **Not checking both parts of TPA.** Getting one part right and one wrong means zero credit.

---

## Key Takeaways

- DS is about sufficiency, not solving — use the elimination framework
- MSR requires reading all tabs before answering any question
- Table questions reward systematic scanning — don't eyeball it
- Graph questions often allow estimation — save time by not over-calculating
- Two-Part Analysis parts are connected — solve one to constrain the other
- Budget 2:00-2:30 per question, max 3:00
